# 🚀 專案分析與計畫總覽：企業餐飲訂單管理系統

本專案是一個基於 \*\*React (前端)\*\* 與 \*\*Firebase (後端/無伺服器架構)\*\* 的全端應用程式，旨在解決企業內部午餐與飲料的團購管理問題。

## 🏗️ 1. 系統架構計畫 (System Architecture)

### 🔹 前端架構 (Front-End)
\*   \*\*技術核心\*\*：React 18.2.0, Tailwind CSS。
\*   \*\*路由管理\*\*：基於 State (\`currentView\`) 的簡易路由控制 (SPA)。
\*   \*\*狀態管理\*\*：
    \*   \`AuthContext\`：全域處理使用者登入狀態、角色權限 (RBAC)、Loading 狀態。
    \*   \`useAuth\` Hook：提供各組件存取使用者資訊與權限檢查函數 (\`isAdmin\`, \`isSuperAdmin\`)。
\*   \*\*服務層 (Service Layer)\*\*：將商業邏輯與 UI 分離。
    \*   \`orderService.js\`：處理訂單建立、查詢、狀態更新。
    \*   \`storeManagementService.js\`：處理店家的 CRUD 與即時監聽。
    \*   \`roleService.js\`：處理使用者角色分配與權限檢查。
    \*   \`imageUploadService.js\`：處理 Firebase Storage 的圖片上傳與壓縮。
\*   \*\*UI 組件模組化\*\*：
    \*   \*\*前台\*\*：\`StoreCardList\` (店家列表), \`MobileOptimizedMenu\` (點餐/客製化), \`OrderHistory\` (歷史紀錄)。
    \*   \*\*後台\*\*：\`StoreManagement\` (店家管理主控台), \`StoreEditor\` (店家編輯), \`MenuEditor\` (菜單編輯), \`OrderStatistics\` (報表統計)。

### 🔹 後端與資料架構 (Back-End / Data)
\*   \*\*平台\*\*：Firebase (Serverless)。
\*   \*\*身份驗證 (Authentication)\*\*：
    \*   Google OAuth 整合。
    \*   \*\*特殊機制\*\*：針對手機版瀏覽器 (Line/FB/Chrome App) 實作 \`signInWithRedirect\` 並搭配 \`localStorage\` 標記 (\`pendingAuth\`) 來解決重導向丟失狀態的問題 \[1\]\[2\]。
\*   \*\*資料庫 (Firestore NoSQL)\*\*：
    \*   \*\*\`users\`\*\*：使用者基本資料。
    \*   \*\*\`userRoles\`\*\*：獨立的角色權限集合 (Security)。
    \*   \*\*\`stores\`\*\*：分層結構 \`stores/{type}/list/{storeId}\`，區分 \`lunch\` (午餐) 與 \`drinks\` (飲料)。
    \*   \*\*\`orders\`\*\*：扁平化儲存所有訂單，包含完整的 Snapshot (當時的價格、名稱、客製化選項)。
\*   \*\*儲存 (Storage)\*\*：
    \*   存放店家封面圖與商品圖片，路徑結構為 \`stores/{type}/{storeId}/...\`。

## 📊 資料分析總結

### 前端資料流
1.  \*\*使用者\*\*：Login -> AuthContext (LocalStorage Check) -> App (Router) -> Home.
2.  \*\*店家管理\*\*：StoreManagement -> Fetch Collections (Lunch/Drinks) -> Merge & Sort -> UI.
3.  \*\*訂單建立\*\*：User Selects Options -> Local Cart State -> \`createOrder\` (Validation + Anti-Dup Check) -> Firestore.

### 後端資料流
1.  \*\*寫入\*\*：主要透過前端 SDK 直接寫入 (Client-side logic)，依賴 Firestore Rules 進行安全驗證。
2.  \*\*讀取\*\*：使用 \`onSnapshot\` 進行即時監聽 (店家列表、訂單狀態)，確保多人協作時資料一致。

這份計畫表涵蓋了從基礎架構到進階功能的完整路徑，您目前已經完成了 \*\*約 85%\*\* 的核心業務功能，剩下的主要是批次處理工具與更深度的使用者互動功能。
